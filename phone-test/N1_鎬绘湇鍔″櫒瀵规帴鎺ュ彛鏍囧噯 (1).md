# N1 盒子 - 总服务器对接接口标准

> 文档版本：v1.2
> 更新日期：2026-04-10
> N1 IP：192.168.5.101 (WiFi) / 192.168.5.18 (有线)
> API基础URL：`http://192.168.5.101`

---

## 一、接口概述

### 1.1 端口分配

| 服务 | 端口 | 协议 | 用途 |
|------|------|------|------|
| Moonraker | 7125 | HTTP | G-code指令下发、心跳检测 |
| go2rtc | 1984 | HTTP | **按需取帧**（MJPG 1280x720） |

### 1.2 拍照方案

**go2rtc按需取帧**：
- 配置：exec模式调用ffmpeg读取v4l2
- 优势：请求时才读取设备，单次取帧后释放资源，CPU消耗极低
- URL：`GET http://192.168.5.101:1984/api/frame.jpeg?src=camera0`

## 二、Moonraker API (端口 7125)

### 2.1 心跳检测

**接口**：`GET /server/info`

**请求**：
```bash
curl http://192.168.5.101:7125/server/info
```

**响应**：
```json
{
  "result": {
    "klippy_connected": true,
    "klippy_state": "ready",
    "moonraker_version": "v0.8.0",
    "api_version": "v1",
    "components": [...],
    "registered_directories": ["config", "logs", "gcodes"]
  }
}
```

**状态判断**：
- `klippy_connected: true` + `klippy_state: ready` → 设备正常，可下发指令
- `klippy_connected: true` + `klippy_state: error` → 固件错误，检查硬件
- `klippy_connected: false` → 未连接，检查Klipper服务

---

### 2.2 发送G-code指令

**接口**：`POST /printer/gcode/script`

**请求**：
```bash
curl -X POST "http://192.168.5.101:7125/printer/gcode/script" \
  -H "Content-Type: application/json" \
  -d '{"script":"G28"}'
```

**常用G-code指令**：

| 指令 | 说明 | 示例 |
|------|------|------|
| `G28` | 归零所有轴 | `{"script":"G28"}` |
| `G1 Z30 F3000` | 移动Z轴到30mm | `{"script":"G1 Z30 F3000"}` |
| `G1 X75 Y75 F6000` | 移动XY到指定位置 | `{"script":"G1 X75 Y75 F6000"}` |
| `M112` | 紧急停止 | `{"script":"M112"}` |
| `FIRMWARE_RESTART` | 重启固件 | `{"script":"FIRMWARE_RESTART"}` |

**注意**：
- `F`参数单位为 **mm/min**，非 mm/s
- F9000 = 150mm/s， F4800 = 80mm/s

---

### 2.3 查询设备状态

**接口**：`GET /printer/objects/query`

**请求**：
```bash
curl "http://192.168.5.101:7125/printer/objects/query?print_stats=state,message"
```

**响应**：
```json
{
  "result": {
    "status": {
      "print_stats": {
        "state": "standby",
        "message": "",
        "filename": "",
        "total_duration": 0.0,
        "print_duration": 0.0,
        "progress": 0.0
      }
    },
    "eventtime": 12345.678
  }
}
```

**state状态说明**：
| 状态 | 说明 |
|------|------|
| `standby` | 空闲，可接收新指令 |
| `printing` | 正在执行脚本 |
| `paused` | 暂停 |
| `error` | 执行错误 |

---

### 2.4 完整API列表

| 接口 | 方法 | 说明 |
|------|------|------|
| `/server/info` | GET | 服务器信息 |
| `/server/restart` | POST | 重启Moonraker |
| `/server Fever RESTART` | POST | 重启Klipper固件 |
| `/printer/gcode/script` | POST | 发送G-code |
| `/printer/objects/list` | GET | 可查询对象列表 |
| `/printer/objects/query` | GET | 查询对象状态 |
| `/printer/emergency_stop` | POST | 紧急停止 |
| `/machine/reboot` | POST | 重启系统 |
| `/machine/shutdown` | POST | 关机 |

## 三、截图服务 API (端口 1984)

> 使用go2rtc按需取帧，无需额外服务

### 3.1 触发拍照

**接口**：`GET /api/frame.jpeg?src=camera0`

**请求**：
```bash
curl -o /tmp/screenshot.jpg "http://192.168.5.101:1984/api/frame.jpeg?src=camera0"
```

**响应**：
- 成功：返回JPEG图片文件 (1280x720)
- 失败：返回错误信息

**注意**：go2rtc使用exec模式，每次请求时启动ffmpeg读取一帧，CPU消耗极低

---

## 四、完整通信流程

### 4.1 自动脚本执行流程

```
1. 总服务器 → GET http://192.168.5.101:1984/api/frame.jpeg?src=camera0
   ↓
2. N1返回JPEG图片 (1280x720)
   ↓
3. 总服务器 → YOLO推理 → 像素坐标 → 机械坐标
   ↓
4. 总服务器 → POST http://192.168.5.101:7125/printer/gcode/script
   ↓
5. N1执行G-code → 轮询 /printer/objects/query 确认完成
   ↓
6. 执行完毕 → 记录日志
```

### 4.2 手动脚本执行流程

```
1. WEB前端编辑G-code → 提交到总服务器
   ↓
2. 总服务器校验脚本（超程/速度检查）
   ↓
3. 总服务器 → POST http://192.168.5.101:7125/printer/gcode/script
   ↓
4. 轮询状态确认执行完成
   ↓
5. WebSocket推送结果到前端
```

### 4.3 急停流程

```
1. 前端点击急停 → 二次确认
   ↓
2. 总服务器 → POST http://192.168.5.101:7125/printer/emergency_stop
              或发送 G-code: M112
   ↓
3. 记录急停状态到数据库
   ↓
4. 人工复位 → POST /api/v1/devices/{id}/reset
```

## 五、错误处理

### 5.1 网络错误

| 错误 | 处理 |
|------|------|
| 连接超时 | 重试1次，失败则标记设备离线 |
| HTTP 500 | 记录错误，跳过当前任务 |
| 拍照失败 | 重试1次，失败则告警 |

### 5.2 执行错误

| 状态 | 处理 |
|------|------|
| `state: error` | 自动抬Z到安全高度，触发告警 |
| 超时60秒 | 标记异常，触发告警 |
| 急停触发 | 停止所有操作，等待人工复位 |

## 六、附录：go2rtc配置

### A.1 go2rtc配置

**路径**：`/etc/go2rtc/go2rtc.yaml`

```yaml
api:
  listen: ":1984"

streams:
  camera0:
    - exec:ffmpeg -f v4l2 -input_format mjpeg -video_size 1280x720 -i /dev/video1 -frames:v 1 -f image2pipe -
```
