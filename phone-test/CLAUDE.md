# CLAUDE.md - 手机APP自动化测试系统

> **交付级别**：特斯拉超级工厂项目，稳定性第一，质量优先。
> **文档用途**：AI编程助手上下文文件，AI读取后应能独立实现整个项目。

## 项目背景

**用户场景**：手机厂商测试APP性能——模拟真实用户操作手机，验证APP运行流畅度、稳定性、匹配性能。

**核心约束**：
- 物理真实测试（不能使用ADB，必须用真实手指/电容笔接触屏幕）
- Android 和 iOS 手机都要支持
- 每天测试目标：10万台（远期扩展目标，当前先跑通22台）
- 团队：3人，30-50天开发周期
- 预算：2万元（硬件已有，主要是人力成本）
- 设备上万台时必须全自动运行，人工不参与干预

**当前状态**：硬件3台样机（N1盒子+CoreXY机械臂+RP2040下位机+C270摄像头），没有跑通任何流程。

### 软件架构

```
总控服务器（3060显卡）
  ├── 交互层：自然语言控制 / HTTP API / WEB界面
  ├── AI控制层：自然语言理解 → 任务拆解 → YAML流程生成
  ├── 视觉层：GUI-Owl-1.5-8B（本地主力）→ API模型（兜底）
  ├── 坐标层：4点标定（像素→机械坐标映射）+ 人工微调
  ├── 控制层：Moonraker API → N1 → RP2040，设备锁（单机独占）

  ├── 调度层：22台并发调度
  └── WEB层：React + MUI（Material UI），暗色主题
```

### 视觉层

**模型**：GUI-Owl-1.5-8B（INT8量化），阿里mPLUG团队GUI Agent专精模型，基于Qwen2.5-VL + 环境RL（MRPO），MIT许可。

| 项目 | 值 |
|------|-----|
| 显存 | ~9GB INT8（3060 12GB可运行） |
| 推理 | ~300ms/图 |
| 部署 | vLLM，OpenAI-compatible API |
| GUI定位 | ScreenSpotPro 80.3（开源SOTA） |
| 手机自动化 | AndroidWorld 71.6（开源SOTA级） |
| 复杂决策 | OSWorld 56.5（开源SOTA），有thinking模式 |
| 成本 | 免费（本地GPU），API兜底 $0.003/图 |

**降级策略**：GUI-Owl-1.5-8B → API模型（GPT-4o/Claude Vision）

**VisionAdapter 统一接口**（实现类：`GUIOwlAdapter`、`APIVisionAdapter`）：

- `find_icon(screenshot, icon_name)` → (x, y) 像素坐标或 None
- `find_element(screenshot, element_desc)` → (x, y) 或 None
- `detect_page_state(screenshot, description)` → bool（如"APP已加载完成"）
- `verify_action(before_screenshot, after_screenshot, action_desc)` → bool
- `read_text(screenshot, region?)` → 文字列表及位置
- `detect_anomaly(screenshot)` → 异常描述或 None
- `plan_actions(screenshot, task_desc)` → 动作序列

**扩展原则**：通过VisionAdapter接口调用，降级顺序可配置，API密钥环境变量配置。

## 技术规范

### 运动参数

| 参数 | 值 |
|------|-----|
| CoreXY行程 | X 0-150mm，Y 0-150mm，Z 0-50mm |
| 安全高度 | Z30mm |
| XY最大速度 | 150mm/s = F9000 |
| Z最大速度 | 80mm/s = F4800 |
| 定位精度 | ±0.5mm |

### 坐标转换

4点标定线性映射：`Mx = (Px / W) * 150`，`My = (Py / H) * 150`，支持人工微调偏移量。

### 设备状态机

```
ONLINE ←→ SUSPECT ←→ OFFLINE → RECOVERING → ONLINE
                                ESTOP（急停，需手动复位）

- ONLINE：心跳正常，可下发脚本
- SUSPECT：单次心跳超时（防WiFi抖动）
- OFFLINE：连续3次超时，触发告警
- RECOVERING：等待连续2次心跳成功
- ESTOP：需手动复位
```

## 流程模板格式

```yaml
app_name: 抖音
vision:
  primary: guiowl
  fallback_order: [guiowl, gpt-4o]
  confidence_threshold: 0.8
steps:
  - action: tap_icon
    icon: 抖音
    wait: 2
  - action: detect_state
    expected: 首页已加载，有推荐视频流
    timeout: 5
  - action: swipe_up
    start_percent: [50, 80]
    end_percent: [50, 20]
    duration_ms: 500
```

**动作类型**：tap_icon / detect_state / verify_action / detect_anomaly / tap / swipe_up / swipe_down / swipe_left / swipe_right / long_press / wait

模板存放：`templates/{app_name}.yaml`，换APP只换模板文件。

## 自然语言控制

用户说指令 → AI解析意图 → 检查标定 → 生成YAML → 执行 → 实时反馈 → 可保存为模板

| 用户说 | AI动作 |
|--------|--------|
| 打开抖音点一下设置 | tap_icon(抖音) → find_element(设置) → tap |
| 向上滑10次，每次等1秒 | swipe_up × 10, interval=1s |
| 这台手机连续刷抖音1小时 | loop(swipe_up, duration=3600s) |
| 暂停所有设备 | emergency_stop(all) |

## 开发规范

**禁止事项**：
1. N1禁止运行计算/识别/决策代码
2. 禁止跳过脚本安全校验
3. 禁止无二次确认的批量操作
4. 禁止急停后自动恢复
5. 禁止同一设备并发下发脚本
6. 禁止使用同步HTTP库（必须aiohttp）

**安全规范**：优先抬Z轴→走XY→下降执行，所有坐标/速度必须校验范围上限，失败优先自动抬Z。

**代码规范**：async/await异步编程，aiohttp通信，日志含设备ID/时间戳/动作/结果，关键操作写入数据库。
